## 1. Introduction

본 프로젝트는 딥페이크 탐지기가 의존하는 지배적 주파수 대역을 내재적으로 교란하는 얼굴 이미지를 합성하기 위해, ControlNet을 Stable Diffusion에 통합한 생성 파이프라인인 **Freq-ControlNet**을 제안한다.

FreqDebias (CVPR 2025)는 딥페이크 탐지기가 위조 유형별 특정 주파수 대역에 과도하게 의존하는 스펙트럼 편향(Spectral Bias)을 실험적으로 규명하였다. 예를 들어 Deepfakes 위조 유형에서는 저주파, NeuralTextures에서는 고주파 대역이 탐지의 주요 단서로 작용한다. 본 프로젝트는 이 발견을 역으로 활용한다. 즉, 탐지기가 의존하는 지배적 주파수 대역을 공간적 조건 맵으로 변환하고 ControlNet을 이로 학습시킴으로써, 생성 이미지의 주파수 특성이 실제 이미지 분포에 근접하도록 Stable Diffusion을 유도하여 탐지기가 의존하는 단서를 제거하는 것이다.

전체 파이프라인은 아래 네 단계로 구성된다.

1. 탐지기 로드 및 주파수 마스크 B 추출
2. 조건 맵 생성 (Type-A / Type-B)
3. ControlNet 학습
4. 이미지 생성 및 평가

## 2. Motivation & Problem Statement

### 2.1 배경: 딥페이크 탐지기의 스펙트럼 편향

딥페이크 탐지기와 생성 모델은 끊임없는 군비 경쟁(arms race)을 벌이고 있다. FreqDebias (CVPR 2025)는 딥페이크 탐지기가 위조 유형별 특정 주파수 대역에 과도하게 의존하는 스펙트럼 편향을 실험적으로 규명하였다. 구체적으로, 탐지기는 Deepfakes에서는 저주파, Face2Face에서는 중간 주파수, NeuralTextures에서는 고주파에 주로 의존하며, 이 편향된 대역이 탐지 결정의 주요 근거가 된다.

### 2.2 핵심 가설

탐지기가 의존하는 지배적 주파수 대역이 생성 단계에서 교란된다면, 탐지기가 의존하는 단서 자체가 생성 이미지에서 사라져 탐지를 자연스럽게 회피할 수 있다.

### 2.3 Research Question

> 주파수 마스크를 ControlNet의 조건으로 주입할 경우, Stable Diffusion이 탐지기의 지배적 대역에 해당하는 교란된 주파수 특성을 가진 이미지를 생성하도록 유도할 수 있는가?

### 2.4 기술적 도전

Canny 엣지나 깊이 맵과 같은 기존 ControlNet 조건과 달리, 주파수 도메인 마스크를 공간 도메인으로 변환하여 도출된 조건 맵은 육안으로는 노이즈나 블러처럼 보이거나 거의 검은 이미지에 가깝다. ControlNet의 제로 컨볼루션 레이어가 이 미세한 주파수 신호를 의미 있는 조건으로 학습할 수 있는지 여부가 이 PoC(개념 검증)가 검증하고자 하는 핵심 질문이다.

## 3. Proposed Method / Technical Details

### 3.1 데이터셋

| 데이터셋 | 종류 | 규모 | 용도 |
|---|---|---|---|
| CelebA-HQ (Align&Cropped) | 실제 이미지 | 5,000장 | ControlNet 학습 |
| FF++ c23 (Deepfakes) | 위조 이미지 | 200클립 × 20프레임 → 2,000장 샘플링 | 주파수 마스크 추출 |
| FF++ c23 (Face2Face) | 위조 이미지 | 200클립 × 20프레임 → 2,000장 샘플링 | 주파수 마스크 추출 |
| FF++ c23 (NeuralTextures) | 위조 이미지 | 200클립 × 20프레임 → 2,000장 샘플링 | 주파수 마스크 추출 |
| DeepfakeBench Xception | 탐지기 가중치 | FF++ HQ 학습 | OHEM 손실 계산 및 평가 |

CelebA-HQ는 유명인 얼굴 사진에서 선별된 실제 이미지로 구성된다. Align&Cropped 버전을 사용하여 정면 얼굴, 균일한 구도라는 일관된 학습 분포를 확보하였다. FF++는 FaceForensics++ 데이터셋으로, c23 압축(시각적으로 거의 무손실)된 비디오에서 클립당 20프레임을 균등 추출하였다.

### 3.2 STEP 1: 주파수 마스크 B 추출

#### 3.2.1 탐지기 준비

DeepfakeBench에서 제공하는 Xception 탐지기(FF++ HQ로 학습된 사전학습 가중치)를 로드한다. 가중치 키 구조가 `backbone.XXX` 형식으로 저장되어 있어, timm의 `legacy_xception` 모델에 맞게 키 변환(`backbone.` 제거, `last_linear.` → `fc.` 치환)을 수행한 후 로드하였다.

#### 3.2.2 반경 기준 Ring 분할

초기에는 FFT 진폭 스펙트럼의 평균값, 반경(r), 각도(θ)를 feature로 하는 K-means(k=8) 클러스터링을 시도하였다. 그러나 다음 두 가지 문제가 발견되었다.

**문제 1 — amplitude 영향력 열세:** K-means 입력 feature에서 `r_n`, `theta_n`의 std(0.20, 0.29)에 비해 `amp_n`의 std(0.054)가 4~5배 작아, 클러스터 모양이 이미지 내용과 무관하게 위치(r, θ)로만 결정되었다.

**문제 2 — 평균 스펙트럼의 유형 간 차이 소멸:** 2,000장의 평균 FFT 진폭 스펙트럼을 비교하였을 때 위조 유형 간 MAE가 0.0017에 불과하여, 2,000장 평균화 과정에서 위조 특유의 국소적 아티팩트가 평균화되어 사라지고 모든 자연 이미지가 공유하는 1/f 분포만 남은 것으로 분석된다.

이에 따라 K-means를 제거하고 **반경(r) 기준 동심원 Ring 8분할**로 전환하였다. Ring 0(중심, 저주파) ~ Ring 7(외곽, 고주파)로 나누고, 클러스터 모양은 모든 유형에 대해 동일한 동심원으로 고정한다. 유형 간 차이는 다음 OHEM 단계에서만 반영된다.

#### 3.2.3 OHEM(Online Hard Example Mining) 기반 Ring 선택

각 Ring을 억제(amplitude → 0)했을 때 Xception 탐지기의 평균 분류 손실을 계산한다. baseline(억제 없음) 대비 손실 변화(Δloss)가 가장 큰 Ring top-3를 선택한다. Δloss가 크다는 것은 해당 Ring을 제거했을 때 탐지기가 더 크게 혼란을 겪는다는 의미로, 탐지기가 그 주파수 대역에 상대적으로 많이 의존함을 나타낸다.

#### 3.2.4 실험 결과 및 한계

| 위조 유형 | baseline 손실 | 최저 손실 Ring | 선택 Ring |
|---|---|---|---|
| Deepfakes | 0.940 | Ring 1 (Δ=-0.310) | [0, 1, 2] → 저주파 |
| Face2Face | 0.919 | Ring 1 (Δ=-0.299) | [0, 1, 2] → 저주파 |
| NeuralTextures | 0.942 | Ring 0 (Δ=-0.302) | [0, 1, 2] → 저주파 |

세 유형 모두 Ring [0, 1, 2](저주파, 전체 스펙트럼의 22%)가 선택되어 유형 간 차별화가 이루어지지 않았다. 이는 Xception의 입력 다운샘플링(299×299)이 고주파 정보를 제한하고, c23 압축이 위조 유형별 고주파 아티팩트를 약화시킨 결과로 분석된다. 본 PoC에서는 세 유형 공통으로 저주파 마스크 B(Ring [0,1,2], 22%)를 확정하여 사용하였다.

### 3.3 STEP 2: 조건 맵 생성

CelebA-HQ 5,000장 각각에 대해 아래 두 종류의 조건 맵을 생성한다. 목표 이미지(target)는 항상 원본 그대로이며, 조건 맵만 달라진다.

#### Type-A (억제 맵)

저주파 Ring [0,1,2]의 amplitude를 0으로 제거하고 IFFT로 재구성한다. 단, DC 성분(Ring 0의 중심 1픽셀, 전체 평균 밝기에 해당)은 보존하여 이미지가 완전히 검게 변하는 것을 방지한다.

```
x'_A = IFFT[ A(x) ⊗ (1 − B') , P(x) ]
B' = B − {DC 픽셀}  (DC 보존)
```

결과: 저주파 디테일(전반적인 형태, 부드러운 음영)이 제거된 흐릿한 에지 패턴 이미지.

#### Type-B (교체 맵)

이미지 A의 저주파 Ring amplitude를 랜덤 페어 이미지 B의 것으로 교체한다. 페어는 같은 5,000장 내에서 seed=42로 무작위 할당하며, 자기 자신과 페어되지 않도록 처리한다.

```
x'_B = IFFT[ A(x_B) ⊗ B + A(x_A) ⊗ (1 − B) , P(x_A) ]
```

결과: 원본과 구조는 비슷하지만 전반적인 명암/색감 분포가 다른 사람 것으로 바뀐 이미지.

#### 검증

pixel-level 차이(mean diff)로 두 타입의 변환이 실제로 적용되었는지 확인하였다.

| 비교 | mean diff | max diff |
|---|---|---|
| target vs Type-A | 77.7 / 255 | 255 |
| target vs Type-B | 28.6 / 255 | 189 |
| Type-A vs Type-B | 76.0 / 255 | 255 |

Type-A는 저주파 제거로 인한 큰 변화, Type-B는 저주파 교체로 인한 중간 수준의 변화가 확인된다.

### 3.4 STEP 3: ControlNet 학습

#### 학습 설정

| 항목 | 값 |
|---|---|
| Base model | runwayml/stable-diffusion-v1-5 |
| ControlNet 초기화 | SD UNet 인코더 가중치에서 복사 |
| 학습 데이터 | CelebA-HQ 5,000장 (조건 맵 + 원본 쌍) |
| Optimizer | AdamW, lr=1e-5 |
| Batch size | 2 (gradient accumulation=2, 실효 배치=4) |
| Steps | 6,000 (≈ 5 epoch) |
| Mixed precision | fp16 |
| GPU | NVIDIA RTX 2080 Ti (11GB VRAM) |
| 학습 시간 | 약 5시간 14분 (Type-A), 동일 (Type-B) |

ControlNet-A와 ControlNet-B를 각각 독립적으로 학습하였다.

#### Loss 추이

| Step | ControlNet-A loss |
|---|---|
| 2,000 | 0.102 |
| 4,000 | 0.240 |
| 6,000 | 0.137 |
| 최종 | 0.156 |

loss가 0.1~0.24 범위에서 진동하며 어느 정도 수렴하는 양상을 보인다.

### 3.5 STEP 4: 이미지 생성

학습된 ControlNet을 사용하여 세 가지 실험 조건으로 각 500장씩 이미지를 생성한다.

| 실험 | 조건 | 파이프라인 |
|---|---|---|
| Exp-0 (베이스라인) | 없음 | SD v1.5 단독 |
| Exp-1 | Type-A 조건 맵 | ControlNet-A + SD v1.5 |
| Exp-2 | Type-B 조건 맵 | ControlNet-B + SD v1.5 |

생성 설정: DDIM 50 스텝, guidance_scale=7.5, controlnet_conditioning_scale=1.0, 해상도 512×512.

#### 초기 정성적 관찰 (10장 샘플)

- **Exp-0**: SD 특유의 렌더링된 완벽한 피부, 스튜디오 조명 느낌의 이미지 생성.
- **Exp-1/2**: CelebA-HQ(실제 사진)를 조건으로 학습한 영향으로 자연스러운 배경, 실제 사진에 가까운 조명 및 구도. ControlNet이 주파수 기반 조건을 학습했음을 시사하는 긍정적 결과.

### 3.6 평가 계획

생성된 이미지 500장을 아래 세 가지 축에서 평가한다.

| 평가 축 | 지표 | 방법 |
|---|---|---|
| 시각적 품질 | FID | 생성 500장 vs CelebA-HQ 500장 기준 비교 |
| 주파수 거리 | 지배적 대역 진폭 MAE | 생성 이미지의 Ring [0,1,2] 진폭 분포 vs CelebA-HQ 실제 이미지 비교 |
| 탐지 회피율 | Evasion Rate | 사전학습 Xception 탐지기가 생성 이미지를 "실제(real)"로 분류하는 비율 |

Exp-0 → Exp-1 → Exp-2 순서로 비교하여 주파수 조건 주입이 탐지 회피에 기여하는지 검증한다.

### 3.7 한계 및 논의

**마스크 유형 간 차별화 실패:** c23 압축과 Xception의 입력 다운샘플링 특성으로 인해 세 위조 유형(Deepfakes, Face2Face, NeuralTextures) 간 선택되는 주파수 Ring이 모두 동일하게 나타났다. 원논문(FreqDebias)이 c0(무압축) 데이터와 픽셀 단위 통계 기반 분석을 사용한 것과 달리, 본 실험은 압축된 데이터와 평균 스펙트럼 기반 분석을 사용하여 유형 간 차이가 평균화 과정에서 소멸되었다.

**Type-A의 극단성:** DC 성분을 보존하였음에도 불구하고 Type-A 조건 맵은 저주파 디테일이 크게 제거된 흐릿한 이미지로, ControlNet 학습의 난이도가 높은 조건이다.

**학습 규모:** 6,000 step(약 5 epoch)은 논문에서 제안한 20 epoch 대비 축소된 학습으로, 더 많은 학습이 이루어질 경우 결과가 개선될 가능성이 있다.
